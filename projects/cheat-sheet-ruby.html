Ruby

Easily-accessible, object-oriented programming

Ruby is an object-oriented programming language derived from C that boasts dynamic typing, automatic semicolon insertion, scores of powerful built-in methods, and more. Ruby is distinguished by how easy it is to write and read, because it often provides several different ways to write the same expressions and it fills in all those pesky details that other languages require you to spell out ( like semicolons and parentheses ).

Ruby Syntax

Semicolon insertion is the most visually obvious difference between Ruby and other C-derived languages. Expressions in Ruby are typically separated by starting a new line: the compiler knows to add in semicolons at the end of each line, where appropriate. 

# Ruby

my_num = 3346

def is_fibonacci?( num )
  fib_nums = [ 0, 1 ]
  fib_nums.each_index do |i|
    if num == fib_nums[ i ]
      return true
    elsif num > fib_nums[ i ]
      if i > 0
      fib_nums.push( fib_nums[ i ] + fib_nums[ i - 1 ] )
      end
    else
    return false
    end
  end
end
				
# Ruby with semicolons

my_num = 3346;

def is_fibonacci?( num )
  fib_nums = [ 0, 1 ];
  fib_nums.each_index do |i|
    if num == fib_nums[ i ]
      return true;
    elsif num > fib_nums[ i ]
      if i > 0;
      fib_nums.push( fib_nums[ i ] + fib_nums[ i - 1 ] );
      end
    else
    return false;
    end
  end
end

You can always include semicolons if you want to ensure nothing goes wrong, or if you want to write several expressions on the same line ( although too many on the same line will look ugly ): Ruby won't punish you for it.

Ruby Objects

Ruby has many of the objects you'll find in related languages: the string, integer, float, and array. Ruby also has a unique object: the hash. A hash is much like an array, but instead of predetermined indexes, you use a unique 'key' and assign it a 'value', creating a 'key-value pair'. Ruby also has two different integer objects: 'Fixnum' and 'Bignum'; 'Fixnum' is the default, but if your integer is too big, Ruby will convert it to a 'Bignum', alotting more memory to store it.
